# 个股详情页重构设计

日期：2026-05-12

## 背景

当前个股详情页 `frontend/src/views/StockDetail.vue` 已能展示涨停基础信息、分时走势、涨停时间线、五档盘口和大单成交，但页面信息分散，图表能力偏弱。用户希望详情页更接近专业盯盘工具，支持日 K、月 K、缩放、多股叠加、涨停 K 变色等功能，同时保持页面清爽。

本设计基于已确认的视觉方向：顶部核心状态，左侧大 K 线区，右侧盘口和大单，下方涨停时间线与核心数据。

## 目标

- 将详情页从“信息堆叠”改为“行情分析工作台”。
- 主图支持分时、日 K、周 K、月 K 切换。
- 主图支持缩放、十字提示、成交量副图。
- 支持叠加指数或个股走势，使用不同颜色区分。
- 涨停 K 线、涨停成交量和时间线事件使用醒目的红色高亮。
- 保留现有五档盘口、大单成交、涨停原因、封板状态等业务信息。
- 优先用项目现有 `echarts` 依赖快速落地，避免一次性引入过重图表体系。

## 非目标

- 不在第一阶段实现完整 TradingView 或通达信级别画线系统。
- 不在第一阶段实现真实 Level2 全量逐笔回放。
- 不重做全站导航和其他页面视觉系统。
- 不引入需要账号或外部付费服务的行情源。

## 推荐方案

第一阶段保留 ECharts，重构 `StockDetail.vue` 的布局和图表配置：

- ECharts 已在项目中使用，接入成本最低。
- ECharts 支持 `candlestick`、`line`、`bar`、`dataZoom`、多 grid 和 tooltip，可覆盖本次核心需求。
- 单根 K 线可通过数据项 `itemStyle` 做涨停变色。
- 叠加个股或指数可先用百分比归一化曲线展示，避免不同价格区间挤压主图。

第二阶段如果继续增强专业图表能力，再评估替换为 KLineCharts 或 HQChart：

- KLineCharts 更适合专业 K 线、指标和覆盖层扩展。
- HQChart 更接近 A 股终端能力，但接入和适配成本更高。
- TradingView Lightweight Charts 轻量且性能好，但 A 股特性需要自行补齐。

## 页面结构

### 顶部核心状态

顶部一行展示：

- 股票名称、代码、市场。
- 当前价格、涨跌幅、封单、换手率。
- 连板、封板状态、题材、首封时间、开板次数等标签。
- 自选按钮保留，但不抢占主信息层级。

顶部区域只放“进页面就必须知道”的结论，不再使用大面积描述表格。

### 主图区域

左侧主图是页面核心，占据最大宽度。

功能：

- 周期切换：分时、日 K、周 K、月 K。
- 图层切换：MA、成交量、叠加走势、涨停高亮。
- 缩放：底部 `dataZoom`，并支持工具按钮放大和缩小。
- tooltip：展示开高低收、涨跌幅、成交量、成交额、叠加对象涨跌幅。
- 涨停变色：涨停 K 线使用深红色，普通上涨红色，下跌蓝色。
- 叠加对象：默认可叠加上证指数或同题材个股，后续支持输入股票代码叠加。

图表分为两个 grid：

- 上方主 grid：K 线、MA、叠加曲线。
- 下方副 grid：成交量柱。

### 右侧盯盘栏

右侧固定信息栏用于实时判断封板质量。

模块：

- 五档盘口：突出涨停价、买一封单、卖一是否为空。
- 大单成交：保留现有阈值逻辑，突出主动买入和涨停价成交。
- 可扩展 Tab：五档、L2、分价。第一阶段只展示已实现的五档和大单入口，L2/分价入口隐藏到后续阶段。

### 下方业务信息

下方分两类：

- 涨停时间线：首封、开板、回封、尾盘加封等事件横向卡片展示。
- 核心数据：题材、行业、成交额、流通市值、涨停原因等紧凑展示。

涨停时间线比描述表更重要，应优先展示；描述性字段放在较低层级。

## 数据设计

### 前端状态

`StockDetail.vue` 需要维护：

- `period`：`timeline | day | week | month`
- `overlays`：叠加对象列表，例如指数或其他股票。
- `chartData`：当前周期 K 线或分时数据。
- `timelineData`：涨停状态变化。
- `orderBook`：盘口。
- `bigOrders`：大单。
- `stockInfo`：涨停详情。

### 接口

现有接口继续使用：

- `GET /api/v1/limit-up/{stock_code}` 获取涨停详情。
- `GET /api/v1/market/{stock_code}/timeline` 获取分时。
- `GET /api/v1/market/{stock_code}/orderbook` 获取盘口。
- `GET /api/v1/market/{stock_code}/big-orders` 获取大单。

需要新增或扩展：

- `GET /api/v1/market/{stock_code}/kline?period=day|week|month&limit=250`
- `GET /api/v1/market/compare?symbols=000001.SH,603893.SH&period=day&limit=250`

第一阶段若后端暂时没有本地历史 K 线，可从东方财富接口按需获取并格式化，返回统一结构：

```json
{
  "stock_code": "603893",
  "period": "day",
  "data": [
    {
      "date": "2026-05-12",
      "open": 96.1,
      "close": 103.42,
      "low": 95.6,
      "high": 103.42,
      "volume": 560000,
      "amount": 1820000000,
      "change_pct": 10.0,
      "is_limit_up": true
    }
  ]
}
```

叠加走势返回归一化涨跌幅曲线，前端直接绘制：

```json
{
  "symbol": "000001.SH",
  "name": "上证指数",
  "data": [
    { "date": "2026-05-12", "change_pct_from_start": 1.2 }
  ]
}
```

## 错误处理

- 涨停详情失败：显示空态并保留返回入口。
- K 线失败：主图显示“暂无 K 线数据”，盘口和大单仍可显示。
- 叠加对象失败：保留主图，只在叠加控制处提示失败对象。
- 盘口和大单刷新失败：保留上一次数据，使用轻量提示，避免频繁弹错。
- 非交易日或无数据日：沿用后端日期回退逻辑，界面显示实际数据日期。

## 实现边界

第一阶段只改这些区域：

- `frontend/src/views/StockDetail.vue`
- `frontend/src/api/market.ts`
- `frontend/src/types/market.ts`
- `backend/app/api/v1/market.py`

需要新增后端测试：

- `backend/tests/test_market_kline_api.py`

不改动列表页、看板页、统计页和全局布局，除非构建失败要求最小修复。

## 测试策略

- 前端构建：运行 `npm run build`。
- 后端测试：新增或更新 K 线接口相关测试。
- 手工验证：打开一个涨停股详情页，确认周期切换、缩放、叠加、涨停变色、盘口刷新、大单过滤都可用。
- 空态验证：使用无 K 线数据或无大单数据的股票，确认页面不崩溃。

## 验收标准

- 个股详情首屏不再出现大块描述表格堆叠。
- 主图区域能切换分时、日 K、周 K、月 K。
- 主图有成交量副图和缩放条。
- 至少支持叠加一个指数或个股，并使用不同颜色展示。
- 涨停 K 线和涨停时间线事件有明显高亮。
- 盘口、大单、涨停时间线仍保留且层级清晰。
- `npm run build` 通过。
