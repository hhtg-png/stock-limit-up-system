# 通达信插件九阳公社归类源设计

## 背景

通达信黑底看盘插件 v1 已经具备涨停播报、异动解析、板块强度、聚合快讯等入口，但涨停播报的板块/概念归类仍主要依赖“复盘网个股涨停原因 + 同花顺 F10 概念 + 本地规则推断”。这个方式能覆盖部分个股，但对目标站相同口径的板块分组命中率不够。

已登录查看 `https://www.jiuyangongshe.com/action` 后确认，该页面更接近目标口径：它先按交易日生成板块分组，再把个股、解析标题、解析正文放入分组下。页面还提供“涨停简图”图片化汇总，可用于人工对照，但生产解析应优先使用结构化列表。

## 目标

本次设计只解决通达信插件的“个股归类与异动原因”来源问题：

- 将九阳公社 `/action` 作为涨停播报和异动解析的优先归类源。
- 归一化输出 `stock_code -> plate / concepts / reason_title / reason_detail / article_url`。
- 继续保留复盘网、同花顺 F10、现有本地数据作为兜底。
- 不把用户登录态、cookie、token 或第三方私有接口凭据写入代码库。
- 不以 duanxianxia 目标站的数据作为生产数据源，目标站仅用于黑盒对照。

## 非目标

- 不复制九阳公社页面、图片、会员内容或第三方付费功能。
- 不在本次设计里改造整个通达信插件 UI。
- 不实现自动代用户登录、扫码、绕过登录或保存用户浏览器会话。
- 不把“涨停简图”图片 OCR 成生产主数据；图片只作为校准参考。

## 数据源设计

新增 `JiuyangongsheActionAttributionProvider`，封装九阳公社 action 归类源。

提供者职责：

- 使用交易日查询 `/action` 对应数据。
- 获取板块分组：例如 `公告`、`地产基建`、`消费`、`电力`、`AI硬件`、`被动元件`、`PCB`、`戴尔`、`半导体`、`医疗医药`、`ST板块`。
- 展开每个分组，提取组内股票代码、解析文章链接、解析标题和解析正文。
- 将解析标题按 `+` 拆分成候选概念，保留标题原文作为 `reason_title`。
- 输出统一的 `PublicStockAttribution` 结构，供现有 `TdxPluginService` 复用。

授权策略：

- 本地和线上通过环境变量或后台配置提供可选授权信息。
- 代码只读取运行时配置，不提交、打印或持久化授权值。
- 缓存只保存归一化后的板块、概念、标题、正文摘要、文章 URL 和更新时间，不保存 cookie 或 token。
- 授权失效时返回 `source_status.jygs_action = "error"` 并自动使用兜底源。

## 合并优先级

归类结果按以下优先级合并：

1. 九阳公社 action 板块分组作为 `plate`。
2. 九阳公社 action 解析标题拆分后的关键词作为 `concepts` 首选。
3. 九阳公社 action 解析正文作为 `reason_detail`。
4. 复盘网涨停原因补充 `reason_title`，仅在九阳公社无标题时使用。
5. 同花顺 F10 概念补充 `concepts`，仅在九阳公社概念不足时使用。
6. 现有本地规则兜底，保证页面不空白。

`target_reason_summary` 的生成规则调整为：

- 有九阳公社分组时，显示 `plate + 首个非重复 concept`。
- 如果首个 concept 与 plate 相同，只显示 plate 或选第二个非重复 concept。
- 没有九阳公社数据时，保持现有复盘网/同花顺推断逻辑。

## 缓存和失败处理

缓存按交易日保存：

- 交易时段内短缓存，避免频繁请求第三方页面。
- 收盘后长缓存，匹配九阳公社页面 `12:00` 和 `15:30` 的更新节奏。
- 缓存命中时仍返回 `updated_at` 和 `is_cache`。
- 数据源失败时页面展示最近缓存和来源告警，不返回空白结构。

失败降级顺序：

1. 当日九阳公社缓存。
2. 复盘网 + 同花顺 F10。
3. 本地涨停原因和既有规则。

## 接口影响

后端对外 API 不新增必需参数：

- `GET /api/v1/tdx-plugins/limit-up-live`
- `GET /api/v1/tdx-plugins/stock-move/{code}`
- `GET /api/v1/tdx-plugins/ths-move/{code}`

现有返回结构继续包含：

- `items`
- `updated_at`
- `source_status`
- `is_cache`
- `warnings`

新增字段只作为兼容扩展：

- `source_status.jygs_action`
- `source_status.jygs_action_cache`
- `item.reason_detail`
- `item.source_url`

前端展示仍使用现有 `target_reason_summary`，无需先改 UI。

## 校准流程

新增或扩展校准脚本：

- 输入我方 `limit-up-live` 输出。
- 输入九阳公社 action 归类输出。
- 输入目标站黑盒可见样本。
- 输出股票级差异：缺失股票、板块不同、概念不同、解析标题不同、排序不同。

目标命中率先看两个指标：

- 股票归类命中率：`plate` 是否一致。
- 展示命中率：`target_reason_summary` 是否与目标站板块列一致。

## 测试计划

后端单元测试：

- 解析九阳公社 action 分组 fixture，得到板块和个股映射。
- 解析文章标题 `A+B+C`，得到去重概念。
- 九阳公社有数据时覆盖复盘网/同花顺推断。
- 九阳公社失败时回退复盘网/同花顺。
- 授权配置缺失时不抛异常，返回明确 source status。

服务测试：

- `limit-up-live` 输出包含九阳公社来源状态。
- 缓存命中时 `is_cache` 和 `updated_at` 正确。
- 对比工具能输出板块差异和展示差异。

回归测试：

- 现有涨停列表、每日资讯、每日分析、杰哥交易模式不受影响。
- 通达信插件股票点击仍走通达信联动，不跳项目内详情。

## 实施顺序

1. 为九阳公社 action 解析器写 fixture 和失败测试。
2. 实现 `JiuyangongsheActionAttributionProvider`。
3. 增加组合提供者，让九阳公社优先、复盘网/同花顺兜底。
4. 接入 `TdxPluginService.get_limit_up_live` 和个股异动接口。
5. 增加校准脚本或扩展现有 compare API。
6. 本地跑后端单元测试、前端结构测试和一次真实页面对比。

## 风险

- 九阳公社 action 完整内容需要登录，授权失效会导致实时归类回退。
- 第三方页面字段可能变化，因此解析器必须 fixture 化并在失败时告警。
- 归类目标和 duanxianxia 目标站不一定完全同源，仍需要持续校准权重。
- 不能把登录态写进代码或日志，否则存在安全和合规风险。

## 验收标准

- 有授权时，涨停播报优先显示九阳公社板块分组。
- 无授权或授权失效时，页面不空白，并清楚显示来源告警。
- `000090` 等此前缺失/不准确个股能从九阳公社 action 得到归类和解析。
- 本地对照报告能列出九阳公社、我方、目标站三方差异。
- 后端相关测试全部通过。
